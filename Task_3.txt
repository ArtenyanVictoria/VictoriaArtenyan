a) Вместо хранения суммы в вершине храним текущий максимум на пути до неё.

if max(current_max, edge_weight) < vertex_max[v]:
    vertex_max[v] = max(current_max, edge_weight)
    priority_queue.push(v, vertex_max[v])

б) Вместо суммы храним произведение.
Замечание:
Если есть веса < 1, алгоритм не сработает, так как добавление ребра может уменьшить стоимость пути (нарушается монотонность).

new_cost = current_product * edge_weight
if new_cost < vertex_product[v]:
    vertex_product[v] = new_cost
    priority_queue.push(v, new_cost)

в) НЕТ
Конкатенация строк не является монотонной операцией и нет гарантии, что добавление ребра не ухудшит путь.

г) Вместо минимизации суммы максимизируем минимальное ребро в пути.
Используем максимизирующую очередь (приоритет по убыванию).

new_min = min(current_min, edge_weight)
if new_min > vertex_min[v]:
    vertex_min[v] = new_min
    priority_queue.push(v, -new_min)  # минус для max-кучи

д) В каждой вершине храним: changes[v] — минимальное число смен цвета до v, distance[v] — сумма длин при этом числе смен.
Приоритет в очереди: сначала по changes, затем по distance.

if (new_changes < changes[v]) or
    (new_changes == changes[v] and new_distance < distance[v])):
    changes[v] = new_changes
    distance[v] = new_distance
    priority_queue.push(v, (new_changes, new_distance))
